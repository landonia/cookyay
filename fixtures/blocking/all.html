All blocking cases (combined)

This single page exercises every declarative blocking pattern in one place. Used by E2E tests that need to verify all cases without navigation.

1. Blocked inline script (analytics)

type="text/plain" data-category="analytics" — inline content, no src.

Inline analytics: blocked

2. Blocked external src script — analytics (GA4 stub)

type="text/plain" data-category="analytics" src="/fixtures/stubs/ga4.js"

GA4 stub: blocked

3. Blocked external src script — marketing (Meta Pixel stub)

type="text/plain" data-category="marketing" src="/fixtures/stubs/pixel.js"

Meta Pixel stub: blocked

4. Blocked iframe — marketing (YouTube-style)

data-src="…" data-category="marketing" — no src, so no network request fires.

YouTube-style iframe: blocked

5. Undeclared-category script (stays blocked forever)

data-category="advertising" — not in the known set; stays blocked and emits a console.warn.

Undeclared script: blocked (expected)
