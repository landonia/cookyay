# Security Policy

## Reporting a Vulnerability

Please do not open a GitHub issue for security vulnerabilities.

Report security issues by email to the address in the npm package's `author` field, or via GitHub's private vulnerability reporting feature (Security → Report a vulnerability).

We aim to acknowledge reports within 72 hours and provide a fix or mitigation plan within 14 days for confirmed vulnerabilities.

## Scope

- Cookyay banner library (`cookyay` npm package)
- `@cookyay/scanner` CLI

## Out of scope

- Vulnerabilities in site owners' consent configurations (the library is a tool; how it's configured is the integrator's responsibility)
- Theoretical compliance gaps without a concrete exploit
