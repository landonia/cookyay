## Implementation notes
- **Parallelizable:** the ~30 services are independent data entries. This is a
  strong candidate for parallel Agent subagent dispatch — e.g. split by category
  (marketing pixels / analytics / functional & video) so subagents author
  disjoint slices of `services.yaml`, then merge. Each slice must pass the 002
  validator independently.
- Ground every signal in the SME report; prefer multi-signal entries (host +
  cookie, or host + path) so they reach `high` confidence under the 006 rule.
- Use `requestPaths` (004) for path-discriminated services (Meta `/tr`, reCAPTCHA
  `/recaptcha/`). Use `scriptUrlGlobs`/`iframeSrcGlobs` so 007 can emit markup.
